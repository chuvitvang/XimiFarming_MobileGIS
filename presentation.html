📋 Tổng Quan Dự Án

XimiFarming MobileGIS là một ứng dụng di động tiên tiến được xây dựng trên nền tảng Android (Java), thiết kế đặc biệt để phục vụ ngành nông nghiệp thông minh. Ứng dụng kết hợp công nghệ định vị địa lý (GIS), xử lý ảnh vệ tinh, và trí tuệ nhân tạo để giúp nông dân quản lý đất đai hiệu quả và theo dõi sức khỏe cây trồng một cách khoa học.

💾 Thiết Kế Cơ Sở Dữ Liệu Không Gian
4.1 Cơ sở dữ liệu Cục bộ (Room SQLite)
SQLite không hỗ trợ kiểu dữ liệu không gian phức tạp, nên dữ liệu hình học được lưu dưới dạng JSON Text:
📍 Plot (Thửa đất): Tọa độ ranh giới được lưu trong coordinatesJson dạng mảng JSON các đỉnh đa giác
🌱 Crop (Cây trồng): Vị trí được lưu trong hai trường latitude và longitude 
Ví dụ: Plot coordinatesJson
[
  {"latitude": 10.5, "longitude": 105.3},
  {"latitude": 10.5, "longitude": 105.4},
  {"latitude": 10.6, "longitude": 105.4},
  {"latitude": 10.6, "longitude": 105.3}
]
4.2 Cơ sở dữ liệu Cloud (Supabase PostgreSQL + PostGIS)
Khi đẩy lên cloud, PostgreSQL sử dụng extension PostGIS để hỗ trợ kiểu dữ liệu không gian thực thụ:
🔲 Bảng Plots: Trường geom kiểu geometry(Polygon, 4326)
🔵 Bảng Crops: Trường geom kiểu geometry(Point, 4326) 
Chuyển đổi Polygon trên Cloud
ST_GeomFromText(
  'POLYGON((105.3 10.5, 105.4 10.5, 105.4 10.6, 105.3 10.6, 105.3 10.5))',
  4326
)
4.3 Trigger Tự động Tính Diện Tích
Cơ chế trigger trên cloud tự động tính diện tích thửa đất (m²) bằng công thức địa lý WGS84:
SQL Trigger PostGIS
CREATE OR REPLACE FUNCTION calculate_plot_area()
RETURNS TRIGGER AS $$
BEGIN
    NEW.area_sq_meters := ST_Area(NEW.geom::geography);
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_calculate_area
BEFORE INSERT OR UPDATE ON plots
FOR EACH ROW
EXECUTE FUNCTION calculate_plot_area();
4.4 Row Level Security (RLS)
Mỗi hàng dữ liệu có cột owner_id liên kết với ID người dùng. RLS đảm bảo mỗi người chỉ nhìn thấy dữ liệu của mình: 
RLS Policy
CREATE POLICY "Allow owners to manage plots"
ON plots FOR ALL
TO authenticated
USING (owner_id = auth.uid())
WITH CHECK (owner_id = auth.uid());

6.1 Quá Trình Đồng Bộ
Bước 1 - PULL: Tải Plots & Crops từ Cloud về máy
Bước 2 - PUSH: Đẩy bản ghi có cờ isSynced=false lên Cloud
Bước 3 - ID Translation: Chuyển ID cục bộ sang ID thực tế của Cloud
Bước 4 - FK Update: Cập nhật khóa ngoại của các bản ghi con
6.2 Thuật Toán Dịch Chuyển ID (ID Translation)
Vấn đề: Room tạo khóa tự tăng (ví dụ: oldId=1), nhưng Supabase tạo khóa khác (newId=159). Nếu xóa bản ghi cũ, cascade delete sẽ xóa toàn bộ dữ liệu con! 
Giải pháp: Transaction an toàn
db.runInTransaction(() -> {
    // 1️⃣ Lưu danh sách đối tượng con
    List<Crop> crops = db.cropDao().getCropsForPlot(oldPlotId);
    
    // 2️⃣ Tạo Plot mới với ID từ Cloud
    Plot newPlot = new Plot(plot.getName(), ...);
    newPlot.setId(newSupabaseId);
    newPlot.setSynced(true);
    db.plotDao().insert(newPlot);
    
    // 3️⃣ Cập nhật FK của toàn bộ con
    for (Crop c : crops) {
        c.setPlotId(newSupabaseId);
        db.cropDao().update(c);
    }
    
    // 4️⃣ Xóa Plot cũ (bây giờ không có con)
    db.plotDao().delete(plot);
});
✅ Kết quả: Toàn vẹn dữ liệu được đảm bảo, không mất dữ liệu con khi đồng bộ.
🛰️ Tích Hợp Google Earth Engine (GEE)
7.1 Xác Thực OAuth2 qua Service Account
Thay vì yêu cầu từng nông dân đăng nhập Google, ứng dụng sử dụng Service Account:
📄 Service Account Key: Tệp service_account_key.json chứa Private Key RSA
🔑 JWT Generation: Sinh mã ký JWT, gửi đến https://oauth2.googleapis.com/token
⏰ Token TTL: OAuth2 token có thời hạn 3600 giây 
JWT Flow
Load Service Account Key (RSA Private Key)
    ↓
Generate JWT Signature
    ↓
POST to oauth2.googleapis.com/token
    ↓
Receive OAuth2 Access Token
    ↓
Use token to call GEE API
7.2 Tính Toán NDVI Trung Bình
NDVI (Normalized Difference Vegetation Index) được tính từ ảnh Sentinel-2:
$$\text{NDVI} = \frac{\text{B8 (NIR)} - \text{B4 (Red)}}{\text{B8 (NIR)} + \text{B4 (Red)}}$$
Quy trình GEE: 
GEE Calculation Steps
1️⃣ Lọc chùm ảnh COPERNICUS/S2_SR (Sentinel-2)
   - Khoảng thời gian: 3 tháng qua
   - Khoảng không gian: Polygon thửa đất từ Client

2️⃣ Tính NDVI từ Band 8 & Band 4

3️⃣ Reduce Region bằng ee.Reducer.mean()

4️⃣ Trả về 1 giá trị NDVI trung bình duy nhất
   (ví dụ: 0.65 = cây khỏe mạnh)

🖼️ Thuật Toán ExG (Excess Green Index)
8.1 Công Thức Toán Học
Excess Green Index là chỉ số phổ biến để tách vùng thực vật xanh khỏi nền đất:
$$\text{ExG} = 2 \times g - r - b$$
Trong đó các kênh màu chuẩn hóa:
$$r = \frac{R}{R+G+B}, \quad g = \frac{G}{R+G+B}, \quad b = \frac{B}{R+G+B}$$
Với $R, G, B \in [0, 255]$ là giá trị RGB từ mã màu pixel.
8.2 Logic Phân Tích & Ánh Xạ Bảng Màu
📊 Xử lý Pixel: Duyệt qua từng tọa độ (row, col) của Bitmap ảnh lá cây
🧮 Tính ExG: Áp dụng công thức cho từng pixel
🎨 Chẩn đoán: Ánh xạ giá trị ExG sang màu 
ExG Analysis & Diagnosis
// Trích xuất pixel từ Bitmap
int[] pixels = new int[bitmap.getWidth() * bitmap.getHeight()];
bitmap.getPixels(pixels, ...);

for (int i = 0; i < pixels.length; i++) {
    int pixelColor = pixels[i];
    
    // Tách RGB
    int r = Color.red(pixelColor);
    int g = Color.green(pixelColor);
    int b = Color.blue(pixelColor);
    
    // Chuẩn hóa
    int sum = r + g + b;
    double norm_r = (double) r / sum;
    double norm_g = (double) g / sum;
    double norm_b = (double) b / sum;
    
    // Tính ExG
    double exg = 2 * norm_g - norm_r - norm_b;
    
    // Chẩn đoán
    if (exg > 0.1) {
        // Lá khỏe mạnh → Xanh lá cây (#3DDC84)
        outputPixels[i] = Color.parseColor("#3DDC84");
    } else {
        // Lá bệnh → Đỏ (#FF0000)
        outputPixels[i] = Color.parseColor("#FF0000");
    }
}

✨ Kết Luận

XimiFarming MobileGIS là một dự án tiên tiến kết hợp các công nghệ hiện đại - Android GIS, Cloud PostgreSQL, Satellite Analytics, và Image Processing - để tạo ra một giải pháp toàn diện cho nông nghiệp thông minh.

Điểm mạnh của dự án:

  • Offline-First Architecture: Hoạt động độc lập khi không có mạng
  • Hybrid Storage: Local + Cloud tối ưu hiệu suất và độ tin cậy
  • Advanced Analytics: NDVI (vệ tinh) + ExG (cục bộ) cho chẩn đoán đa chiều
  • Spatial Database: PostGIS cho xử lý GIS tính toán chuẩn xác
  • Security First: Xác thực đầy đủ, RLS, mã hóa dữ liệu

Với kiến trúc vững chắc và công nghệ tiên tiến, XimiFarming MobileGIS sẵn sàng mở rộng quy mô để phục vụ hàng ngàn nông dân, đóng góp vào sự phát triển nông nghiệp bền vững tại Việt Nam.
